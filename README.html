Guia de Setup — Card Kiosk + Hotfolder Printer

Windows 11 + Evolis Dualys • Foto no navegador → job no hotfolder → automatizador imprime. Este guia é focado em você conseguir colocar o fluxo de pé e depurar rápido quando algo der errado.

Hotfolder: C:\card_hotfolder\in Webapp: PHP + JS + Tailwind (CDN) Automatizador: Python (pywin32)
Importante A Dualys é antiga. Se algo “misteriosamente” não funcionar no Windows 11, quase sempre é driver/configuração de spooler/permissão. O webapp só cria jobs; quem decide CR80/duplex/borda é principalmente o driver.

Instruções

Versão: 1.0

0) Pré-requisitos

Obrigatório
  • Windows 11
  • Evolis Dualys conectada e reconhecida
  • Python 3.10+ (recomendado 3.11)
  • PHP 8+ (para subir o webapp local)
  • Chrome/Edge (para câmera)
Recomendado
  • Rodar tudo primeiro “no seu usuário” (sem IIS) para isolar problema de permissão.
  • Só depois mover para serviço/IIS.

1) Conferir a impressora no Windows

  1. Abra Configurações → Bluetooth e dispositivos → Impressoras e scanners.
  2. Confirme que a Dualys aparece e imprima uma página de teste (ou um job simples).
  3. Anote o nome exato como aparece no Windows. Ele precisa bater com o que o automatizador usa.
Preferências do driver (ajuste 1x e esquece)
  • Tamanho: CR80 / ID-1 (85.6 × 53.98 mm)
  • Orientação: a que corresponde ao layout da sua arte
  • Duplex: habilite se você quer frente e verso real
  • Borda/edge-to-edge: habilite se o driver oferecer

2) Criar pastas do hotfolder

Estrutura padrão usada por este fluxo:

PowerShell 
New-Item -ItemType Directory -Force -Path `
  C:\card_hotfolder\in, `
  C:\card_hotfolder\done, `
  C:\card_hotfolder\error, `
  C:\card_hotfolder\logs | Out-Null
O que cada pasta significa
  • in: entrada (o webapp cria jobs aqui)
  • done: processados com sucesso
  • error: falhas (olhar logs + conteúdo do job)
  • logs: log do automatizador

3) Instalar dependências do automatizador (Python)

PowerShell 
py -m pip install --upgrade pip
py -m pip install pillow pywin32 pymupdf
  • pillow: preparar imagens (cortar/encaixar)
  • pywin32: imprimir via Windows GDI/DC
  • pymupdf: aceitar PDFs (se você usar PDF)
Se você não vai imprimir PDF Pode omitir pymupdf, mas qualquer PDF enviado cairá em error.

4) Rodar o automatizador (hotfolder watcher)

4.1) Salvar o script

Salve o script do automatizador em um caminho fixo, por exemplo:

Caminho sugerido 
C:\card_hotfolder\card_hotfolder_printer.py

4.2) Descobrir o nome exato da impressora (opcional)

PowerShell 
py C:\card_hotfolder\card_hotfolder_printer.py --list_printers

4.3) Rodar manualmente (teste)

PowerShell 
py C:\card_hotfolder\card_hotfolder_printer.py --root C:\card_hotfolder --printer "Evolis Dualys"
O que você deve ver Logs no terminal indicando que está monitorando C:\card_hotfolder\in.
Regra de ouro Primeiro rode manualmente e faça todos os testes. Só depois transforme em serviço.

5) Testes do automatizador sem webapp (recomendado)

5.1) Teste com uma imagem (frente apenas)

  1. Copie um arquivo .png ou .jpg para C:\card_hotfolder\in.
  2. O automatizador deve imprimir e mover o arquivo para done.
Se cair em error Abra C:\card_hotfolder\logs\hotfolder.log e leia o motivo exato.

5.2) Teste com job completo (pasta + manifest) para frente e verso

Crie a pasta:

Caminho 
C:\card_hotfolder\in\job_teste_01\

Coloque dentro:

  • front.jpg
  • back.jpg
  • manifest.json
manifest.json 
{
  "printer_name": "Evolis Dualys",
  "copies": 1,
  "duplex": "true",
  "fit_mode": "fill",
  "rotate_degrees": 0,
  "card_size_mm": { "width_mm": 85.6, "height_mm": 53.98 },
  "front_file": "front.jpg",
  "back_file": "back.jpg"
}
Duplex pode virar “dois cartões” Se o driver não estiver realmente configurado como duplex, 2 páginas podem imprimir como 2 cartões separados. Ajuste isso no driver.

6) Subir o webapp (kiosk)

6.1) Ajustar o arquivo de config

Edite src/config.php e confirme:

  • hotfolder_in_path = C:\\card_hotfolder\\in
  • default_printer_name = nome exato da impressora no Windows

6.2) Subir com servidor embutido do PHP

PowerShell 
cd C:\caminho\card_kiosk_webapp
php -S 0.0.0.0:8080 -t public

Abra:

URL 
http://localhost:8080
Health check Na tela inicial o webapp mostra se o hotfolder está OK (é um teste de escrita no caminho configurado).
Câmera Garanta permissão do navegador para câmera. Em kiosk, use Chrome/Edge e dê permissão para localhost.

7) Teste ponta a ponta (webapp → hotfolder → impressora)

  1. Deixe o automatizador rodando em um terminal.
  2. Abra o webapp (http://localhost:8080).
  3. Faça um teste Só frente (captura 1 foto).
  4. Depois teste Frente e verso (captura 2 fotos).
Resultado esperado
  • Webapp cria uma pasta de job dentro de C:\card_hotfolder\in
  • Automatizador processa e move para done
  • Impressora imprime (simples ou duplex conforme driver)
Se “trava” no meio
  • Veja C:\card_hotfolder\logs\hotfolder.log
  • Veja o job em error (se foi movido)
  • Confirme se a impressora está Online e não em pausa

8) Rodar como serviço (produção)

Recomendação prática Use o Agendador de Tarefas (Task Scheduler). É simples, confiável e evita dor de cabeça com serviços customizados.

8.1) Automatizador via Task Scheduler

  1. Abra Task Scheduler → Create Task…
  2. Triggers: At startup (ou At log on)
  3. Actions: Start a program
  4. Program/script: py
  5. Arguments (ajuste o nome da impressora):
Arguments 
C:\card_hotfolder\card_hotfolder_printer.py --root C:\card_hotfolder --printer "Evolis Dualys"
Permissões de serviço Se a tarefa rodar com outro usuário, esse usuário precisa:
  • Escrever em C:\card_hotfolder\*
  • Ter acesso à impressora (spooler)

8.2) Webapp “em produção”

  • Para kiosk local, php -S costuma bastar.
  • Para algo estável: IIS + PHP (FastCGI) ou Apache.
  • Se usar IIS/Apache, vá direto para a seção Permissões.

9) Permissões (causa #1 de falha)

Cenário A: tudo rodando no seu usuário Normalmente funciona sem mexer em ACL.
Cenário B: IIS/Apache/serviço O webserver pode não conseguir escrever em C:\card_hotfolder\in.

9.1) Como corrigir (modo GUI)

  1. Clique direito em C:\card_hotfolder → Propriedades → Segurança
  2. Edite e adicione o usuário/grupo do serviço
  3. Permissão: Modify (Leitura + Escrita + Deletar)
Usuários/grupos comuns
  • IIS: IIS_IUSRS ou identidade do App Pool
  • Apache/XAMPP: usuário do serviço do Apache
  • Task Scheduler: usuário configurado na tarefa
Como testar rápido Abra o endpoint de health do webapp:
  • http://localhost:8080/api/health.php
Se não conseguir escrever no hotfolder, ele retorna erro com o motivo.

10) Diagnóstico rápido

Automatizador não imprime
  • Confirme que o script está rodando
  • Veja hotfolder.log
  • Veja se o job foi para error
Frente e verso vira dois cartões
  • Driver não está realmente em duplex
  • Ajuste nas preferências do driver
Câmera não abre
  • Cheque permissões do navegador
  • Use localhost (mais previsível)
  • Evite IP sem HTTPS em ambientes travados
Onde olhar primeiro
  • Logs: C:\card_hotfolder\logs\hotfolder.log
  • Falhas: C:\card_hotfolder\error
  • Sucessos: C:\card_hotfolder\done

11) Checklist final (antes de dizer “está pronto”)

  • Dualys imprime um teste no Windows (sem o webapp)
  • Nome exato da impressora confirmado (sem typo)
  • Pastas do hotfolder criadas e acessíveis
  • Automatizador processa uma imagem e move para done
  • Teste de pasta + manifest (duplex) entendido e consistente com o driver
  • Webapp health retorna OK (hotfolder gravável)
  • Webapp “Só frente” cria job e imprime
  • Webapp “Frente e verso” cria job e imprime
  • Rodando como serviço (Task Scheduler) sem quebrar permissões

12) Coletar informações para suporte (quando pedir ajuda)

Quando algo falhar, estas informações resolvem 90% dos casos:

  • Nome exato da impressora no Windows
  • Windows 11 build (Configurações → Sistema → Sobre)
  • Trecho relevante do arquivo hotfolder.log
  • Um exemplo completo de job que foi para error (pasta com manifest + imagens)
  • Se estava tentando simplex ou duplex
Dica Se o problema for só “o verso imprime separado”, quase sempre é ajuste de driver/duplex. Se o problema for “webapp diz erro no health”, quase sempre é permissão de escrita no hotfolder.
